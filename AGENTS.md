# AGENTS.md

## 1. 适用范围

本文件适用于 `D:\5CQ项目\5cq_html` 项目。

这是一个以静态页面为主的前端站点项目，主要由以下内容构成：

- HTML 页面
- JS 脚本
- CSS 样式
- 页面模板目录
- 多套历史副本与备份文件

处理本项目时，重点是确认真实生效文件、控制首屏渲染、避免重复渲染、避免把公共配置写散。

## 4. 工作原则

### 4.1 先读页面，再读脚本，再读公共层

建议顺序：

1. 当前页面 HTML
2. 当前页面对应 JS
3. `statics/js/` 下公共逻辑
4. `statics/css/` 下公共样式
5. `statics/config.js`

### 4.2 最小改动

默认遵循：

- 只改当前问题相关页面
- 只改当前问题相关脚本
- 尽量保持现有 DOM 结构
- 不把本来独立的数据项错误合并
- 不因为顺手整理扩大改动面

### 4.3 公共配置优先集中

涉及以下内容时，优先检查或集中到公共层：

- 请求地址
- WebSocket 地址
- 用户信息对象
- 页面初始化状态
- 复用常量

不要在多个页面重复硬编码相同地址或相同状态变量。

### 4.4 保持首屏稳定

处理用户中心、任务中心、抽奖、背包等页面时，优先保证：

- 首屏不空闪
- 避免“先空值后补值”
- 避免重复渲染
- 避免旧数据覆盖新数据
- 避免背包或任务面板提前暴露抽奖结果

---

## 5. 推荐排查顺序

### 5.1 页面渲染问题

优先查看：

1. 当前 HTML
2. 对应页面 JS
3. 登录后用户数据注入点
4. WebSocket 更新点
5. 二次渲染或刷新逻辑

### 5.2 首次登录数据闪烁

重点检查：

- 首屏用户信息来源
- 登录完成后对象写入时机
- 页面首次渲染是否早于数据到位
- 是否依赖异步补渲染

### 5.3 公共地址或公共变量问题

优先检查：

- `statics/config.js`
- `statics/js/`
- 页面内是否有重复硬编码

## 6. 数据库操作规则

本项目虽然以前端为主，但如果任务涉及数据库联调、脚本、后端配合或查询验证，仍然必须遵守以下规则。

### 6.1 默认只允许 `SELECT`

在没有用户明确确认之前，只允许只读查询，包括：

- `SELECT`
- 只读统计查询
- 只读联表查询
- `EXPLAIN`

### 6.2 以下操作必须先询问用户

执行前必须先询问并得到明确确认：

- `INSERT`
- `UPDATE`
- `DELETE`
- `REPLACE`
- `ALTER`
- `CREATE`
- `DROP`
- `TRUNCATE`
- `RENAME`
- 迁移脚本执行
- 初始化脚本执行
- 批量修复数据

---

## 7. 文件操作规则

### 7.1 删除文件只能单个删除

任何删除动作都必须满足：

- 一次只能删除一个文件
- 不能批量删除
- 不能删除整个目录
- 不能按通配符清理一组页面

### 7.2 删除文件只能移动到回收站

允许：

- 将单个文件移动到回收站

禁止：

- 永久删除
- 绕过回收站删除
- 批量清理 `副本`、`备份`、`_new` 文件

### 7.3 删除前必须确认

删除前必须先确认：

- 该文件是否真实在线上使用
- 是否被其他页面引用
- 用户是否明确同意

不确认则不删。

---

## 8. Git 与现有改动规则

- 不回滚用户已有改动
- 不覆盖未知来源修改
- 禁止使用高风险清理命令
- 脏工作区下只修改当前任务相关文件

本项目历史文件多、并行改动多，必须特别注意不要误改错误版本页面。

---

## 9. 修改建议

优先处理这些问题类型：

- 首屏空值闪烁
- 重复渲染
- 任务中心二次刷新
- 用户中心数据晚到
- 抽奖与背包时序不一致
- 公共配置散落

前端修复应尽量做到：

- 逻辑集中
- 状态清晰
- 页面少抖动
- 修改范围小

---

## 10. 验证要求

改动完成后，至少检查：

- 页面首屏显示是否稳定
- 是否仍有空值到有值的跳变
- 是否重复渲染
- 是否存在旧数据覆盖
- 公共配置是否被统一引用

如果不能实际运行页面，必须明确说明未验证项。

---

## 11. 强制红线

以下规则必须严格执行：

1. 先确认真实生效文件，再修改。
2. `index.html` 和 `ranking.html` 的对应模板优先改 `_new.html`。
3. 数据库默认只能执行 `SELECT`。
4. 任何写库操作都必须先询问用户。
5. 删除文件只能单个删除。
6. 删除文件只能移动到回收站，不能永久删除。
7. 不得批量清理 `副本`、`备份`、`_new` 文件。
